2021.1.24

面向对象

一个简单的冒泡排序，了解java基础的面向对象具体实现方法

方法的重写

继承关键字 extends

在子类中对于父类中的同名同参数的方法进行覆盖修改，通过新创建的子类对象调用该eat方法时，调用的是“学生吃饭”。

区分方法的重载和重写：
重载是同名方法，不同形参列表

子类不能重写父类中private权限的方法，原方法返回值是基本类型，则重写的方法返回值必须和原来的方法相同。

Super关键字

子类和父类中有同名的，同类型的一个属性，在子类对象中调用父类中的该条属性，使用super关键字，调用子类本身的该名称的属性，使用this关键字（也可以省略）。

同样用于调用父类中被子类重写的同名方法。

多态性

父类对象的引用指向了子类的对象

调用p的子父类同名方法时调用的是子类重写过的方法（Man）。

p不能调用Man类特有的方法。

对象的多态性只适用于方法，不适用于属性。

person调用man特有的属性，使用强制类型转换

instanceof关键字

a instanceof A：判断对象a是否是类A的实例，返回bool值

用来判断是否能强制转换。people->man 彳亍 man->woman 不彳亍

包装类Wrapper

把基本数据类型也包装成了class，提供了方法，更彻底地面向对象。

基本数据类型 包装类 String 三者相互转换

static

所有的对象，某些特定数据在内存中只有一份。如所有的中国人对象中共享一个“国籍”属性。

Static修饰属性，方法，代码块，内部类。

比如两个同类的对象，有一个static的属性，对c1的该属性赋值，c2的也自动获得了值。

单例设计模式

对于某个类只存在一个对象实例

故，访问权限private，只能调用静态方法，对象的变量是静态的

两种单例设计模式

饿汉式

懒汉式

接口

interface定义，与class是结构并列的，接口中有抽象方法&全局常量（JDK7and before）

一个类可以有多个接口

只有当类实现了接口中的所有抽象方法才可以

否则该类还应声明为abstract抽象类。

接口相当于是一种规范

格式

接口与类之间的关系：实现

类和类之间的关系：继承

接口与接口之间的关系：继承 可以多继承

接口的具体使用，体现了多态性。

抽象类abstract 和 接口interface 都不可以实例化。

代理模式

为其他对象提供一种代理以控制对象的访问

线程安全问题

  • 1.问题：出现重复的票号
  • 出现原因：某个线程操作车票的过程中，尚未操作完成时，其他线程参与进来
  • 如何解决：当一个线程在操作ticket时其他线程不能参与，直到操作完成，即使出现了阻塞
  • 同步机制
  • 方式一，同步代码块
  • synchronized(同步监视器){
  • //需要被同步的代码（操作共享数据（ticket）的代码）
  • }
  • 同步监视器，俗称 “锁” 任何一个类的对象都可以充当🔒，多个线程必须用同一把锁
